# DEPLOY CHECKLIST (шпаргалка)

Коротко: чтобы изменения появились на сайте, нужно:
1) изменить код,
2) проверить локально,
3) отправить изменения в GitHub,
4) дождаться деплоя на Render,
5) проверить сайт.

---

## 0) Один раз настроить (если уже сделал, пропусти)

- Проект уже подключен к GitHub.
- Render уже подключен к этому GitHub-репозиторию.
- В Render уже заполнены переменные окружения:
  - `DATABASE_URL`
  - `JWT_SECRET`
  - `JWT_EXPIRES_IN`
  - `PG_SSL`
  - `CORS_ORIGIN`

---

## 1) Внести изменения в код

- Измени нужные файлы в проекте.
- Сохрани файлы (`Ctrl + S`).

---

## 2) Проверить локально (на своем ПК)

Открой терминал в папке `backend` и выполни:

```powershell
cd "c:\Users\msi\Downloads\DIPLOM_24 04 — копия\DIPLOM_4\DIPLOM_22\backend"
npm start
``

Проверка:
- открой `http://localhost:4000/health` (должен быть JSON с `ok: true`);
- открой `http://localhost:4000` и проверь, что твое изменение видно.

Если все хорошо, останови сервер:
- в терминале нажми `Ctrl + C`.

---

## 3) Отправить изменения в GitHub

Открой терминал в корне проекта (`DIPLOM_22`) и выполни:

```powershell
cd "c:\Users\msi\Downloads\DIPLOM_24 04 — копия\DIPLOM_4\DIPLOM_22"
git add .
git commit -m "Короткое описание изменений"
git push
```

Пример сообщения коммита:
- `git commit -m "Fix profile save validation"`
- `git commit -m "Update social feed styles"`

---

## 4) Проверить деплой на Render

1. Открой Render -> свой сервис.
2. Перейди во вкладку `Events` или `Logs`.
3. Дождись статуса `Live` и успешного завершения деплоя.

Если автодеплой отключен:
- нажми `Manual Deploy` -> `Deploy latest commit`.

---

## 5) Проверить сайт в интернете

Открой:
- `https://fitness-life.onrender.com/health`
- `https://fitness-life.onrender.com`

Проверь, что:
- сайт открывается;
- новые изменения видны;
- логин/регистрация работают.

---

## 6) Если что-то сломалось

1. Смотри ошибку в `Render Logs`.
2. Скопируй последние строки ошибки.
3. Пришли их в чат, и я дам точное исправление.

---

## Важные правила

- Изменения не попадают на сайт, пока ты не сделал `git push`.
- Файл `backend/.env` не нужно отправлять в GitHub.
- Секреты (`JWT_SECRET`, `DATABASE_URL`) хранятся в Render Environment Variables.
- На бесплатном Render первый запрос после простоя может быть медленным (это нормально).
